# Kitchen Service Documentation

## Responsibilities
# - Menu management
# - Kitchen order processing
# - Food status updates

# ## Database Schema (MongoDB)
# ### food_menu Collection
# - food_id: Unique identifier
# - name: Food item name
# - quantity: Available quantity
# - availability: Stock status
# - image: Food image URL
# - note: Additional notes
# - category: Food category

# ### kitchen_orders Collection
# - order_id: Unique identifier
# - items: List of ordered items
# - status: Order status

# ## API Endpoints
# - GET /menu → View food menu
# - PUT /menu/{id} → Update food status
# - POST /kitchen_orders → Receive orders from Order Service
# - PUT /kitchen_orders/{id} → Update order status
